# Contributing

## Process

- Create an issue first and assign it to a milestone.
- Create a feature branch with format `fb-<issue-number>`.
- Open a Pull Request to the active release branch (`X.Y.Z`).
- Close the issue only after acceptance criteria are met.
- Milestone is merged to `main` only when complete and runnable.

## Commit style

Use clear messages with issue reference.

Examples:

- `feat(#3): add todo list page`
- `fix(#8): guard deep-link for inactive todo`
- `docs(#12): add project conclusion in README`

## Definition of done per issue

- Code is runnable locally.
- Acceptance criteria are fulfilled.
- Related docs are updated if needed.
- PR links the issue and milestone.
